# MHWs Talisman Analyzer - Functional & UI Specification v3

本ドキュメントは、モンスターハンターワイルズ（MHWs）護石解析ツールのフロントエンド実装における「正解」を定義する。Stitchは実装および修正において、本仕様を最優先事項として遵守すること。

## 1. デザイン・コンセプト
- プロジェクト名称 VISION ENGINE (Wilds Edition)
- スタイル プレミアム・ダーク・インダストリアル
  - ベースカラー `#05070a`, `#0d1117`
  - アクセント アンバー（`#f59e0b`  スキル・強調）、ブルー（`#3b82f6`  スロット・解析）
- 視覚効果 グラスモルフィズム（`backdrop-blur`）、微細な発光（`glow`）、高密度なレイアウト。

## 2. 構造的制約 (Structural Constraints)
- サイドバー禁止 画面左側のサイドナビゲーションは一切排除し、ヘッダーに集約すること。
- 装飾的SF要素の禁止 世界地図、汎用的な統計グラフ、円グラフ等は「ノイズ」とみなし、配置しない。
- バッチ処理フロー 各カードに「承認（ConfirmAccept）」ボタンを置かない。編集はカードクリックによる詳細画面（Manual Override）でのみ行う。

## 3. 画面仕様：Dashboard (一覧画面)
### 3.1 統計パネル (Bento Box)
- 上部に「解析進捗」「検出数」「要確認数」「ファイル名」を4枚のパネルで表示。

### 3.2 タイムライン・カード
- レイアウト 
  - 左側：縦長（34）の護石ステータス画像クロップ。
  - 右側：判定データ。
- レスポンシブ 画面幅に応じて「列数」を増やす（モバイル1、タブレット2、PC 3〜4列）。カード横幅を広げすぎないこと。
- データ表示順 (厳守)
  1. 最上部ヘッダー 信頼度スコア（左）  タイムスタンプ（右）
  2. 1段目 レアリティバッジ（例 RARE 5）
  3. 2段目 スロット構成（Lv値を明記。例 [3] [2] [-]）
  4. 3段目 スキルリスト（名称とLvのセット）

## 4. 画面仕様：Manual Override (詳細モーダル)
### 4.1 構成順序
1. Rarity 抽出スコアを表示し、レア度を手動選択。
2. Slot Recognition Profile 
   - 3スロット分の波形グラフを表示。
   - 重要 各スロットに `S (Std)` と `B (BG)` の数値ラベルを必ず表示すること。
   - Lv（0-4）および種別（Armor  Weapon  None）の選択。
3. Skill Identification OCR
   - OCR元画像（クロップ）を表示。
   - スキル名（セレクトボックス）とLv（数値入力）。
   - 最大3枠。動的な追加・削除ボタンを備える。
4. ROI Live Feed 解析対象領域のビデオプレビューを表示（地図は不可）。

## 5. インタラクション・ロジック
- デジタルズーム カードのサムネイルにホバーした際、右側に「High Fidelity Crop」プレビューをオーバーレイ表示。
- 信頼度警告 個別または総合スコアが `0.90` 未満の場合、警告色（アンバー）とアイコンを適用。
- スキャン演出 ROIモニタ内にレーザースキャンラインのアニメーションを実装。

## 6. 実装上の注意
- イミュータブルな状態管理 ReactのState更新時に深い階層（`slots[i].lv` 等）を直接操作せず、スプレッド演算子等を用いて正しく更新すること。
- セマンティックHTML 外側のコンテナを `div`、メインコンテンツを `main`、サイド要素を `aside` として正しくマークアップすること。
