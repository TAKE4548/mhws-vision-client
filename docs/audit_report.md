# セキュリティ監査レポート (Security Audit Report)

## 1. 監査サマリー (Audit Summary)
本リポジトリに対して、外部プラットフォームへの公開（GitHub等）を想定したセキュリティ監査を実施しました。決定論的なファイル抽出（`git ls-files`）に基づき、ローカルLLM（DeepSeek-R1）およびヒューリスティック・スキャニングを併用して、秘密情報、PII（個人情報）、およびローカルパスの漏洩を確認しました。

**ステータス:** ⚠️ 要修正
主要なリスクとして、開発環境特有の絶対パス（`C:\Users\...`）が複数のファイル（主にドキュメントおよび一部のソースコード）に含まれていることが確認されました。これらは公開前に削除または汎用的な表現に置換する必要があります。

## 2. 発見されたリスク (Detected Risks)

| リスク種別 | 重要度 | 内容 | 影響 |
| :--- | :--- | :--- | :--- |
| ローカルパスの露出 | High | `<WORKSPACE_ROOT>\...` 形式の絶対パスが含まれている | ユーザー名やディレクトリ構造が露出する |
| 秘密情報の露出 | Low | 現時点でハードコードされたAPIキーやパスワードは見つかっていない | 比較的安全だが、継続的な監視が必要 |
| 設定情報の露出 | Medium | テスト計画やタスクログに内部サーバーの構造が詳細に記載されている | システム構成の推測を容易にする |

## 3. 該当箇所とファイルパス (Specific Locations)

| ファイルパス | 行番号 | 種別 | 内容/説明 |
| :--- | :--- | :--- | :--- |
| `src/components/vision/VideoUploader.tsx` | 134 | ローカルパス | プレースホルダーに `C:\Users\...` が例示されている |
| `docs/tasks/REQ-029.md` | 11, 13, 19, 21 | ローカルパス | サーバー側のファイルへの絶対パスが記載されている |
| `docs/design/PBI-REQ-030-test-plan.md` | 12, 13, 26 | ローカルパス | テスト用スクリプトやエビデンス保存先に絶対パスが使用されている |
| `docs/backlog/task/archive/REQ-029.md` | 11, 12, 17, 18 | ローカルパス | アーカイブされたタスク内に絶対パスが残存している |

## 4. 推奨される修正アクション (Recommended Remediations)

1. **絶対パスの置換 (最優先)**:
   - 全ての `<WORKSPACE_ROOT>\...` を `./` や `%USERPROFILE%`、または抽象的な表現（`<workspace_root>` 等）に置換してください。
   - 特に `src/` 配下のソースコードに含まれる例示用のパスは、即座に修正が必要です。

2. **ドキュメントのクリーンアップ**:
   - `docs/` 配下のタスクログやデザインドキュメントから、特定の個人環境に依存する記述を削除してください。

3. **継続的なスキャン**:
   - 今後の開発においても、コミット前にヒューリスティックなスキャン（`grep` による特定パターンの検索）を自動化することを推奨します。
