# System Feature: API Stub Mode (REQ-014)

## 1. 概要
バックエンドサーバーの未実装や停止、不安定な環境に左右されることなく、フロントエンド単体での開発・結合試験を完遂するための内部シミュレーション機能です。

## 2. アーキテクチャ

### Axios Interceptor (短絡路)
- `src/lib/api-client.ts` のリクエスト・インターセプターによって実装されています。
- `uiStore` 内の `apiMode` が `stub` の場合、リクエストを実際のネットワークに投げず、設定に基づいたレスポンスを即時に返却します。

### Mock Data Source
- `src/lib/mock-data.ts` がすべてのスタブデータの唯一の情報源（SSoT）です。
- 各エンドポイントは `openapi.yaml` のレスポンス・スキーマに準拠した形式で定義されています。

## 3. 主要機能

### 動的切り替え
- サイドバーの「API Mode」トグルから、実行時にいつでも `Live` と `Stub` を切り替え可能です。
- 再読み込みなしで即時に通信モードが変更されます。

### エラーシミュレーション
- Stubモード時にのみ有効化できる「Error Mode」を搭載しています。
- 全ての API リクエストが一律で HTTP 500 エラー (`Promise.reject`) を返すようになり、UIのエラーハンドリング（トースト通知、リトライUI等）の堅牢性をテストできます。

## 4. 拡張方法 (モックデータの追加)
新しい API エンドポイントを追加した場合、以下の手順でスタブ対応を行います：

1. `src/lib/mock-data.ts` に新しいモックオブジェクト（例：`MOCK_NEW_DATA`）を定義。
2. `src/lib/api-client.ts` のリクエスト・インターセプター内の分岐ロジックに、対象の URL パターンと定義したモックを紐づける。
3. `wrapMock()` ユーティリティを使用して `CommonResponse` フォーマットに包んで返却。
