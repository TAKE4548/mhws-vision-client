# PBI-024-test-plan: テスト基盤の検証計画

## 1. 単体テスト要求事項
### 1.1 既存ストアテストの実行
- **対象**: `src/store/roiStore.test.ts`
- **期待結果**: `npm test` を実行した際、全てのテストケースが正常にパスすること。
- **確認事項**: Zustand のステート更新が Vitest 環境下で正しく検証できていること。

### 1.2 新規コンポーネントテストの試行
- **対象**: `src/components/common/NumericalAdjuster.tsx` (または同等の単純なコンポーネント)
- **期待結果**:
    - コンポーネントが正しくレンダリングされること。
    - ボタンクリック等のインタラクションにより、期待されるコールバックが呼ばれる、または表示が更新されること。

## 2. 統合・環境検証シナリオ
### シナリオ 1: Vitest グローバル API の有効性
- `describe`, `it`, `expect` を明示的にインポートしなくてもテストが実行できることを確認する（Globals 設定の検証）。

### シナリオ 2: JSDOM 環境の確認
- テスト内で `document` や `window` が定義されており、DOM 操作が可能であることを確認する。

### シナリオ 3: Jest-DOM マッチャーの利用
- `expect(...).toBeInTheDocument()` 等のカスタムマッチャーが動作することを確認する。

- [ ] `npm test` コマンドで `roiStore.test.ts` が動作し、パスする。
