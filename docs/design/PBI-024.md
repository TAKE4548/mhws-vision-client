# PBI-024: VitestおよびReact Testing Libraryによるテスト基盤の構築

## 1. 目的と背景
現在のプロジェクトには Vitest や React Testing Library が導入されておらず、自動テストの実行や TDD による開発が困難な状態です。
本 PBI では、これらのツールを導入し、既存の `roiStore.test.ts` を正常に実行可能にするとともに、コンポーネントテストの基盤を確立します。

## 2. 影響範囲
- `package.json`: 依存関係の追加、テストスクリプトの追加
- `vite.config.ts`: Vitest 設定の統合
- `tsconfig.json`: テスト環境用（jsdom等）の型定義追加
- 新規作成: `src/test/setup.ts` (テスト環境のグローバル初期化)

## 3. 技術的仕様

### 3.1 導入ライブラリ
- **Vitest**: Vite ベースの高速なテストランナー
- **@testing-library/react**: React コンポーネントのテスト用
- **@testing-library/jest-dom**: DOM の状態をアサーションするためのマッチャー拡張
- **jsdom**: Node.js 環境での DOM エミュレーション

### 3.2 構成案
#### `vite.config.ts` の拡張
`defineConfig` に `test` プロパティを追加し、`globals: true`, `environment: 'jsdom'`, `setupFiles: './src/test/setup.ts'` を設定します。

#### `src/test/setup.ts` [NEW]
- `@testing-library/jest-dom` のインポート。
- グローバルなモックが必要な場合の定義（例: `ResizeObserver` 等）。

### 3.3 既存テストの修正
- `src/store/roiStore.test.ts` が Vitest の API (`describe`, `it`, `expect` 等) をインポートせずに動作するよう（globals: true 設定時）、または明示的にインポートするよう調整します。

## 4. ロジック・フロー
1. 必要な開発依存関係のインストール。
2. `vite.config.ts` への Vitest 設定の追加。
3. `src/test/setup.ts` の作成と JS/DOM 環境の初期化。
4. `npm test` スクリプトの実行による `roiStore.test.ts` のパス確認。
5. サンプルコンポーネントテストによる React Testing Library の動作確認。

## 5. 考慮事項
- **TypeScript 型定義**: `vitest/globals` を `compilerOptions.types` に追加する必要があるか検討。
- **MSW 連携**: 既に `msw` が導入されているため、Node.js 環境での MSW サーバー起動設定を `setup.ts` に含めるべきか（将来の API テスト用）。
