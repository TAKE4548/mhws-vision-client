# テスト計画書: ROI完全同期・等間隔調整 (REQ-020)

## 1. 単体テスト (Store Logic)
`src/store/roiStore.ts` のロジックが期待通りに動作することを検証する。

### シナリオ 1: スロット座標の横方向同期
- **操作**: `slot_icon` (id: 0) の `x_rel` を変更。
- **期待結果**: id: 1 と id: 2 の `x_rel` が、(id: 0 の座標 + 幅 + gap) および (id: 1 の座標 + 幅 + gap) に自動更新されること。Y座標、W、Hが id: 0 と一致すること。

### シナリオ 2: スキル座標の縦方向同期
- **操作**: `skill_name` (id: 1) の `y_rel` を変更（子枠の移動）。
- **期待結果**: id: 0 を含むグループ全体が同期して移動し、相対位置（gap）が維持されること。

### シナリオ 3: Gap 調整の反映
- **操作**: `slot_icon` の `gap` 値を変更。
- **期待結果**: id: 0 は動かず、id: 1 と id: 2 の `x_rel` が新しい gap 値に基づいて即座に再計算されること。

## 2. 結合テスト (UI/Interaction)
ブラウザ上で実際の操作感と視覚的フィードバックを検証する。

### シナリオ 4: ターゲット選択の簡略化
- **検証項目**: 右側パネルにおいて、Slot 1/2/3 という個別の選択肢が消え、`SLOT ICONS` 等のカテゴリ単位の選択肢になっていること。

### シナリオ 5: 複数枠のドラッグ
- **操作**: キャンバス上で Slot 2 を掴んで移動させる。
- **期待結果**: Slot 1 と Slot 3 も等間隔を維持したまま、マウスの動きに追従して移動すること。

### シナリオ 6: スライダーによる Gap 微調整
- **操作**: UI スライダーで Gap を変更。
- **期待結果**: キャンバス上の 3 つの枠の間隔がリアルタイムで伸縮すること。

## 3. 境界条件・異常系
- **最小Gap**: Gap をマイナス方向（枠が重なる状態）に設定してもエラーにならないこと。
- **画面端**: 同期移動により、一部の枠が `parent_window` の境界外に出た場合の挙動（クランプまたは警告表示）の確認。
- **プロファイル保存**: 同期後の座標データが `POST /api/v1/config/roi/profiles` に正しく送信され、再読み込み後も配置が再現されること。
