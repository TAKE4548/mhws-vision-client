# PBI-027: ROI同期グループの細分化 (Test Plan)

## 1. 検証シナリオ

### 1-1. 同期グループの独立性確認
- [ ] 「Slot Icons」グループの要素（例: Slot 1 Icon）をドラッグして動かす。
- [ ] Slot 2, 3 の **Icon** が連動して動くことを確認。
- [ ] Slot 1, 2, 3 の **Level Border** が動かないことを確認。

### 1-2. 各グループの連動確認
- [ ] 「Slot Level Borders」の要素を動かし、同グループのみが連動することを確認。
- [ ] 「Skill Names」の要素を動かし、同グループのみが連動することを確認。
- [ ] 「Skill Level Borders」の要素を動かし、同グループのみが連動することを確認。

### 1-3. Gap 調整機能の確認
- [ ] 各カテゴリ選択中、サイドパネルの「Gap Adjustment」が対応する Gap 値を表示していることを確認。
- [ ] スライダーまたは数値入力で Gap を変更し、即座に 3 つの枠の間隔が更新されることを確認。

### 1-4. 同期無効化の確認
- [ ] 「Sync Enabled」トグルを OFF にする。
- [ ] プライマリ要素を動かしても、他の要素が連動しないことを確認。

## 2. 自動テスト (Unit Test)
- `src/store/roiStore.test.ts` を更新または新規作成し、以下の項目を検証する。
  - `syncProfile` が 4 つのグループを正しく、かつ独立して同期すること。
  - `updateGaps` が正しい Gap キーを更新し、対応するグループにのみ影響を与えること。

## 3. 手動テスト (Browser)
- `ROICalibrator` 画面の Step 3 (Item ROIs) にて、実際にマウス操作で各カテゴリを調整し、予期せぬ連動が発生しないか目視で確認する。
