# REQ-028 検証計画

## 1. 自動テスト
- `npx orval` がエラーなく完了することを確認。
- `api-client.ts` の単体テスト（もしあれば）または、モックを用いた `FormData` 送信時のヘッダー動作確認。

## 2. 手動検証（ブラウザ + 実機サーバ）
※ 実機サーバ (`localhost:8000`) が起動している必要がある。

### テストケース 1: 動画アップロードの正常性
1.  ブラウザでアプリを開く。
2.  「SELECT VIDEO」から適当な動画を選択。
3.  ネットワークタブを確認し、`POST /api/v1/analyze/video` のリクエストヘッダー `Content-Type` に `boundary=----...` が含まれていることを確認。
4.  レスポンス `202 Accepted` と `job_id` が返ってくることを確認。

### テストケース 2: SSE 接続と解析開始
1.  アップロード完了後、ROIプロファイルを選択（またはデフォルト）。
2.  「INITIATE ANALYSIS」をクリック。
3.  `POST /api/v1/analyze/start/{job_id}` が正常終了することを確認。
4.  SSE 接続 (`GET /api/v1/analyze/events/{job_id}`) が確立され、イベントがストリームされることを確認（コンソールログ）。

### テストケース 3: エラーハンドリング
1.  サーバを停止した状態でアップロードを試み、適切なエラーメッセージが表示されることを確認。
