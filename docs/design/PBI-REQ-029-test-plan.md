# テスト計画: REQ-029 解析済み護石のキャプチャ画像表示機能の復旧

## 1. 目的
画像表示機能が期待通りに動作し、全ての解析済み護石に対して正しい画像が紐付けられていることを確認する。

## 2. テストケース

### 2.1 自動テスト (バックエンド)
- `VisionService.analyze_frame` が正しいパス（ルートの assets/crops/）にファイルを保存することを確認する。
- API `/talismans` が正しい `image_url` を返すことを確認する。

### 2.2 ブラウザテスト (フロントエンド)
- **ケース1: リアルタイム表示**
  1. 動画解析を開始する。
  2. SSEで `talisman_analyzed` が届いた際、Dashboardの「Live Discoveries」に画像が表示されること。
  3. コンソールに404エラーが出ていないこと。
- **ケース2: 詳細画面**
  1. 解析済みのカードをクリックして詳細モーダルを開く。
  2. モーダル内の「A. Observational Data Extraction」セクションに画像が表示されること。
- **ケース3: リロード後の表示**
  1. 解析完了後にページをリロードする。
  2. 結果一覧から再度画像が表示されることを確認する（DBからの取得確認）。

## 3. 検証環境
- ローカル開発環境 (Vite + FastAPI)
