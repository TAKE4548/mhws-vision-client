# PBI-018: 動画解析プログレスバー表示の異常数値修正 - テスト計画

## 1. ユニットテスト要件
### フロントエンド (Vitest)
- `visionStore.ts` の `listenToEvents` における `progress` イベントの処理をテストする。
- 入力 `0.5` に対して状態が `50` になることを確認。
- 入力 `1.2` (異常値) に対して、クリッピングまたは適切なエラーハンドリングが行われるか検討（現状は 100% を超えないことを保証する）。

### バックエンド (Pytest)
- `task_queue.py` の進捗計算ロジックをテストする。
- 処理フレーム数と総フレーム数から期待される小数 (0.0-1.0) が返されることを確認。

## 2. 統合テスト (E2E)
### シナリオ: 解析進捗の正常表示
1. ユーザーが動画をアップロードし、解析を開始する。
2. プログレスバーが表示される。
3. **期待値**: 数値が 0% から始まり、100% を超えることなく滑らかに増加すること。
4. **期待値**: 1枚目のスキャン時（数秒後）に、1000% などの巨大な数値にならないこと。

## 3. 受け入れ条件 (AC) チェックリスト
| ID | 条件 | 確認方法 |
|---|---|---|
| AC-1 | 表示数値が 100% を超えないこと | ブラウザで動作確認 |
| AC-2 | 1枚目のスキャン時点で 1000% 超にならないこと | ブラウザで解析開始直後の数値を確認 |
| AC-3 | 解析完了時に 100% に到達すること | シーケンス完了後の表示を確認 |
