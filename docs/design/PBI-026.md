# 設計書: ROI操作時のレスポンス向上とレイアウト安定化 (REQ-026)

## 1. 目的
ROI（解析範囲）のドラッグやリサイズ操作中に発生する背景画像の頻繁な再描画（APIフェッチ）を抑制し、操作の軽快さと視覚的な安定性を向上させる。また、画像ロード時のレイアウトシフト（ガタつき）を解消する。

## 2. 変更内容

### 2.1. プレビュー画像フェッチロジックの最適化 (`src/components/ROICalibrator.tsx`)
- **依存配列の整理**: `useEffect` によるプレビュー取得のトリガーを、`step` の変更時、および `activeTarget` が「背景の切り替えが必要なカテゴリ」に変更された時に限定する。
- **操作中の抑制**: 矩形の座標変更（`profile.parent_window` 等）による自動フェッチを停止する。
- **フェッチパラメータの固定**: ステップ遷移時に、そのステップで必要な背景画像パラメータ（全画面、親ROI、またはスロットROI）を確定させ、そのステップ内では同じ背景画像を維持する。

### 2.2. レイアウト安定化の実装 (`src/components/roi/InteractiveCanvas.tsx`)
- **ダブルバッファリング的アプローチ**: 新しい画像がロード完了するまで古い画像を表示し続ける、またはロード中のスケルトン表示への切り替え時のサイズ変更を抑制する。
- **アスペクト比の固定**: `previewImage` が `null` になった際に `actualRatio` を即座にリセットせず、コンテナの `aspectRatio` を維持することで、画像ロード待ちの間のレイアウトシフトを防止する。
- **object-fill の適用**: コンテナサイズと画像のアスペクト比に微差がある場合でも、枠線との整合性を優先して `object-fill` を適用する。

### 2.3. ストアの拡張 (`src/store/roiStore.ts`)
- 特になし。既存のステート管理で対応可能。

## 3. 影響範囲
- `src/components/ROICalibrator.tsx`: `useEffect` 内のフェッチロジック。
- `src/components/roi/InteractiveCanvas.tsx`: 画像表示およびアスペクト比計算ロジック。

## 4. Trade-off Disclosure
- **即時フィードバックの欠如**: 座標を変更した直後に「その範囲の最新画像」が背景に反映されない。しかし、ROI操作は「固定された背景画像に対して枠を合わせる」作業であるため、実用上のデメリットは極めて低い。
- **手動更新の必要性**: 別の動画時間（timestamp）に変更した際、背景画像を更新するには一度 Step 1 (Source) に戻って再キャプチャするか、更新ボタン（今後検討）を押す必要がある。

## 5. 考慮事項
- **ステップ3 (Item ROIs)** において、ステップ2で設定した `parent_window` が背景として使用される。ステップ2から3へ移動した瞬間に一度だけフェッチが走るようにする。
- **ステップ4 (Normalization)** において、スロットの拡大画像が背景として使用される。ステップ3から4へ移動した瞬間に一度だけフェッチが走るようにする。
