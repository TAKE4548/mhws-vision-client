# REQ-032-FE: クライアント・サーバ間接続不備の解消 (FE) 詳細設計

## 1. 目的
ローカル開発環境において、ブラウザによる `localhost` の IPv6 優先名前解決（`::1`）と、サーバー（FastAPI/Uvicorn）の IPv4 待機（`127.0.0.1`）の不一致による接続失敗を解消する。

## 2. 変更内容

### 2.1 API クライアント設定の修正
- **ファイル**: `src/lib/api-client.ts`
- **変更**: `API_HOST` の値を `http://localhost:8000` から `http://127.0.0.1:8000` へ変更。
- **理由**: 名前解決の揺れを排除し、IPv4 ループバックアドレスを直接指定することで確実にバックエンドへ接続するため。

## 3. 影響範囲
- `apiClient` インスタンスを使用する全ての API 通信。
- `resolveImageUrl` 関数による画像 URL 解決。

## 4. 依存関係
- [PBI-SYS-REQ-032](../../../docs/design/PBI-SYS-REQ-032.md) (システム横断設計)
- バックエンド (mhws-vision-server) が `127.0.0.1:8000` で起動していること。
