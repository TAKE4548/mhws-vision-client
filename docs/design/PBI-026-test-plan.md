# 検証計画: ROI操作時のレスポンス向上とレイアウト安定化 (REQ-026)

## 1. 自動テスト (Unit/Integration Tests)
- `src/store/roiStore.ts` の座標更新アクションが、背景画像フェッチの状態に影響を与えないことを確認する（依存関係の分離）。
- Vitest を使用し、不要なレンダリングやフェッチが発生していないかコンポーネントのテスト（Mock API）で検証する。

## 2. 手動検証 (Manual Test Cases)

| ID | 検証項目 | 期待される結果 |
|---|---|---|
| TC-026-01 | 矩形ドラッグ時のパフォーマンス | Step 2 または Step 3 で枠をドラッグした際、ブラウザの Network タブで `/vision/preview` へのリクエストが連発されないこと。 |
| TC-026-02 | 操作の軽快さ | 枠の動きがマウスカーソルに追従し、描画の遅延（カクつき）が感じられないこと。 |
| TC-026-03 | ステップ遷移時の背景更新 | Step 2 から Step 3 へ進んだ際、背景が全画面から「親ROI範囲」に正しく切り替わること。 |
| TC-026-04 | レイアウトシフトの防止 | 背景画像の切り替え時、キャンバスコンテナのサイズが急変したり、白い画面が一瞬表示されたりするガタつきがないこと。 |
| TC-026-05 | 数値入力時の挙動 | NumericalAdjuster で値を変更した際、枠の位置は即座に更新されるが、背景画像のリロードは発生しないこと。 |

## 3. 境界条件・異常系
- ネットワーク遅延がある環境（Stubモードで遅延をシミュレート等）でも、UIがフリーズせずに操作可能であること。
- 画像フェッチが失敗した場合でも、既存の背景画像が維持されるか、適切なプレースホルダーが表示され、レイアウトが崩れないこと。
