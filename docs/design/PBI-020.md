# 設計書: ROI完全同期・等間隔調整 (REQ-020)

## 1. 目的
ROI設定において、ゲームUI上で等間隔に並んでいるスロット（3枠）およびスキル（3枠）を個別に調整する手間を省き、1つの枠の操作で全枠を正確かつ迅速に配置できるようにする。

## 2. 変更内容

### 2.1. データ構造の変更 (`src/store/roiStore.ts`)
- `ROIState` に `gaps` オブジェクトを追加し、各カテゴリの間隔を管理する。
  ```typescript
  gaps: {
    slot_icon: number;
    slot_level: number;
    skill_name: number;
    skill_level: number;
  }
  ```
- `activeId` の役割を縮小。カテゴリ選択時は常にグループ全体を対象とするが、基準点計算のために内部的に保持。

### 2.2. 同期ロジックの実装
`updateRelativeRect` を拡張し、どの枠を動かしてもグループ全体に波及する「全枠強制同期」を実装する。

#### スロット (横方向同期)
- **基準**: 1枠目の左辺（`x_rel`）
- **制約**:
  - `Slot[n].y_rel = Slot[0].y_rel`
  - `Slot[n].w = Slot[0].w`
  - `Slot[n].h = Slot[0].h`
  - `Slot[1].x_rel = Slot[0].x_rel + Slot[0].w + gap`
  - `Slot[2].x_rel = Slot[1].x_rel + Slot[1].w + gap`

#### スキル (縦方向同期)
- **基準**: 1枠目の上辺（`y_rel`）
- **制約**:
  - `Skill[n].x_rel = Skill[0].x_rel`
  - `Skill[n].w = Skill[0].w`
  - `Skill[n].h = Skill[0].h`
  - `Skill[1].y_rel = Skill[0].y_rel + Skill[0].h + gap`
  - `Skill[2].y_rel = Skill[1].y_rel + Skill[1].h + gap`

### 2.3. ユーザーインターフェースのリデザイン (`src/components/roi/ROICalibrator.tsx`)

#### 右側操作パネルの変更
- **ターゲット選択**:
  - `RARITY`
  - `SLOT ICONS` (旧 Slot 1/2/3 Icon を統合)
  - `SLOT LEVELS` (旧 Slot 1/2/3 Level を統合)
  - `SKILL NAMES` (旧 Skill 1/2/3 Name を統合)
  - `SKILL LEVELS` (旧 Skill 1/2/3 Level を統合)
- **Gap 調整**:
  - 選択中のカテゴリに応じた `gap` 値を調整するスライダーおよび数値入力欄を追加。
  - スライダー範囲: `-50` 〜 `100` px (マイナスは重なりを許容)。

#### キャンバス上のインタラクション
- 3つの枠のどれを掴んでドラッグしても、全体の相対関係を維持したまま移動するように `handleMouseMove` を調整。

## 3. 影響範囲
- `src/store/roiStore.ts`: ステート管理および更新ロジック。
- `src/components/roi/InteractiveCanvas.tsx`: ドラッグ時のターゲット座標計算。
- `src/components/roi/ROICalibrator.tsx`: 操作パネルのUIとターゲット選択ロジック。

## 4. 考慮事項
- **初期化**: 既存のプロファイルを読み込んだ際、現在の座標から逆算して `gap` の初期値を設定するか、デフォルト値（0等）を適用するかの判断が必要。→ 初回は現在の配置を維持しつつ、操作が加わった時点で同期を適用する。
