### REQ-026: ROI操作時のレスポンス向上とレイアウト安定化
- **Type**: refinement
- **Status**: done (2026-04-23)
- **Current step**: none
- **Priority**: P1
- **Surface**: ROI操作中の背景再描画による遅延と、レイアウトのガタつきの解消。
- **Root Cause**: Interaction (Workflow friction / Excessive feedback) - 枠の操作という高頻度なイベントに対し、画像フェッチやレイアウト計算を伴う重い再描画が同期的に実行されている。また、再描画時のレイアウト不安定性が操作性を損なっている。
- **Requirement**: ROI操作中の背景画像更新を抑制し、ステップ遷移時のみに限定することで、操作の軽快さを確保する。また、画像ロード中のレイアウト・フリッカーを防止する。
- **Acceptance criteria**:
  - ROI枠（バウンディングボックス）のドラッグ・リサイズ操作中、背景画像は更新されず、枠のみがリアルタイムかつ軽快に動くこと。
  - 背景画像の再描画（およびAPIフェッチ）は、原則としてステップ遷移時のみに限定されること。
  - 画像の更新時、コンテナサイズが一旦デフォルトに戻るようなガタつき（Layout shift）がなく、スムーズに遷移すること。
  - 数値入力による微調整時も、操作のたびに画像フェッチを走らせず、レスポンスを優先すること。
- **Design doc**: none
