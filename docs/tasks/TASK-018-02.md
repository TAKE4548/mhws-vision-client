# TASK-018-02: フロントエンド進捗処理の安全性強化

## 概要
バックエンドから受信した進捗値（小数 0-1）をパーセンテージ（0-100）に変換し、かつ異常値（100%超）が表示されないよう保護ロジックを追加する。

## 対象ファイル
- `mhws-vision-client/src/store/visionStore.ts`

## 実装内容
- `listenToEvents` 内の `progress` イベント受信部で、受信値を `Math.min(data.progress, 1.0) * 100` のように処理する。
- `syncJobStatus` 内のプログレス同期部も同様に `Math.min(data.progress || 0, 1.0) * 100` とする。

## 確認事項
- `npm run test` (Vitest) でプログレス値のクリッピングが動作することを確認。
- スタブモードでの動作に影響がないことを確認（スタブは既に 0-1 を送信しているため）。
