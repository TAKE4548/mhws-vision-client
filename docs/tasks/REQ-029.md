# Task Card: REQ-029 解析済み護石のキャプチャ画像表示機能の復旧

## 1. 概要
バックエンドの画像保存パスの不整合を解消し、DBスキーマを拡張して画像URLを永続化する。フロントエンド側ではURL解決ロジックを共通化・堅牢化する。

## 2. タスク分割

### Task 1: [BACKEND] 保存パス修正とDBスキーマ拡張
- **目的**: 画像が正しい公開ディレクトリに保存されるようにし、DBにURLカラムを追加する。
- **対象ファイル**:
  1. `<WORKSPACE_ROOT>\ws\hogehoge\mhws-vision-server\src\services\vision_adapter.py`
     - `save_path` を `Path("assets/crops")` (プロジェクトルート) に変更。
  2. `<WORKSPACE_ROOT>\ws\hogehoge\mhws-vision-server\src\db\database.py`
     - `TalismanModel` に `image_url: Mapped[Optional[str]] = mapped_column(String, nullable=True)` を追加。

### Task 2: [BACKEND] 保存ロジックの更新とAPIマッピング
- **目的**: 解析時にURLをDBに保存し、APIレスポンスでそれを利用するようにする。
- **対象ファイル**:
  1. `<WORKSPACE_ROOT>\ws\hogehoge\mhws-vision-server\src\services\task_queue.py`
     - `run_analysis_task` 内で `new_talisman` 作成時に `image_url=talisman.image_url` を追加。
  2. `<WORKSPACE_ROOT>\ws\hogehoge\mhws-vision-server\src\api\v1\endpoints\talismans.py`
     - `list_talismans` および `update_talisman` において、DBから取得した `image_url` を優先してレスポンスに含めるように修正。

### Task 3: [FRONTEND] 画像URL解決ロジックの改善
- **目的**: フロントエンド側で画像のURLをより安全に解決し、リトライロジックを共通化する。
- **対象ファイル**:
  1. `src/components/vision/AnalysisMonitor.tsx`
     - 画像表示部分のURL構築ロジックを修正。
  2. `src/components/vision/TalismanDetailsModal.tsx`
     - モーダル内の画像URL解決とエラーハンドリングを修正。

## 3. 技術メモ
- **URL解決ロジック**: `(url) => url.startsWith('http') ? url : `${API_HOST}${url}`` のような形式を基本とする。
- **バックエンドパス**: プロジェクト実行ディレクトリ直下の `assets/crops` が `http://host:port/assets/crops` として配信されている。
