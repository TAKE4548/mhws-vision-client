# TASK-027-01: roiStore.ts の状態定義と同期ロジックの修正

## 概要
ROI同期グループを4つに細分化するため、`roiStore.ts` の `gaps` 定義を拡張し、`syncProfile` ロジックを刷新します。

## 対象ファイル
- `src/store/roiStore.ts`

## 実装詳細
1. `ROIState['gaps']` を以下の4キーに変更する。
   - `slotIconGapX`: number
   - `slotLevelGapX`: number
   - `skillNameGapY`: number
   - `skillLevelGapY`: number
2. `DEFAULT_PROFILE` および `ensureProfileStructure` に適合するように初期値を調整。
3. `syncProfile(profile, gaps)` を修正。
   - **Slot Icons Group**: `slots[0].icon` を基準に `slots[1,2].icon` の `x_rel` を `slotIconGapX` 間隔で同期。
   - **Slot Levels Group**: `slots[0].level` を基準に `slots[1,2].level` の `x_rel` を `slotLevelGapX` 間隔で同期。
   - **Skill Names Group**: `skills[0].name` を基準に `skills[1,2].name` の `y_rel` を `skillNameGapY` 間隔で同期。
   - **Skill Levels Group**: `skills[0].level` を基準に `skills[1,2].level` の `y_rel` を `skillLevelGapY` 間隔で同期。
   - **重要**: スロットのアイコンとレベル枠の間の連動（現在は `primary.icon.x_rel + gaps.levelGapX` のように書かれているもの）を削除し、完全に独立させる。
4. `updateRelativeRect` 内で `syncProfile` を呼び出す際、意図した通りに動作することを確認。

## 完了条件
- 4つの独立した Gap が定義され、それぞれが正しく同期ロジックに反映されていること。
