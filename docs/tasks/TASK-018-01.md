# TASK-018-01: バックエンド進捗値の小数統一 (0.0-1.0)

## 概要
動画解析タスクの進捗報告（SSEイベントおよびDB保存値）を、パーセンテージ (0-100) から小数 (0.0-1.0) に変更する。これにより OpenAPI 仕様およびフロントエンドの期待値と整合させる。

## 対象ファイル
- `mhws-vision-server/src/services/task_queue.py`

## 実装内容
- `run_analysis_task` 内の `progress` 計算ロジックから `* 100.0` を削除する。
- 最終完了時の `progress=100.0` を `progress=1.0` に変更する。
- 初期化時の `progress=0.0` はそのまま。
- `min(..., 99.0)` 等のクリッピングを `min(..., 0.99)` に変更する。

## 確認事項
- `pytest` で `task_queue` の進捗計算が 1.0 を超えないことを確認。
