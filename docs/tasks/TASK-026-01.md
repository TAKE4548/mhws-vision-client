# Task: ROICalibrator.tsx のプレビューフェッチ最適化 (TASK-026-01)

## 概要
ROI操作中の頻繁な API フェッチを抑制するため、`useEffect` の依存配列を整理し、背景画像の更新が必要なタイミングのみ実行されるようにします。

## ターゲットファイル
- `src/components/ROICalibrator.tsx`

## 実装内容
1. `useEffect` (L87付近) の依存配列から `profile.parent_window`, `profile.resolution`, `activeTarget`, `activeId` を削除します。
2. プレビューの背景として使用される「カテゴリ」を以下のロジックで算出します：
   - `FULL_FRAME`: `step` が `setup`, `source`, `parent` の場合。
   - `PARENT_WINDOW`: `step` が `items` の場合。
   - `SLOT_PREVIEW`: `step` が `normalization` の場合。
3. この「カテゴリ」が変更された時、または `step`, `jobId`, `timestampMs`, `selectedProfileId` が変更された時のみフェッチを実行するようにします。
4. フェッチ開始時の `setPreviewImage(null)` (L95) を削除（またはオプション化）し、ロード完了まで旧画像を維持するようにします。

## 期待される挙動
- 枠をドラッグしたり、NumericalAdjuster で値を変更したりしても、Network タブにリクエストが発生しないこと。
- ステップを切り替えた瞬間に、新しい背景画像のリクエストが発生すること。
